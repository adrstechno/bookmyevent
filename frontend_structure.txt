Frontend Application Structure & Regeneration Guide
=================================================

1) Short Overview (kya hai)
- Ye project React + Vite frontend hai jisme Tailwind + MUI use hua hai. App ek event/booking dashboard aur public-facing pages provide karta hai (listing, services, booking flow, reviews, auth/OTP).
- Entrypoint: `index.html` -> `src/main.jsx` (Vite + React mount). Bundler: Vite. Package scripts in `package.json`.

2) Tech stack
- Framework: React (with Vite)
- Styling: Tailwind CSS + MUI (Material UI)
- Tooling: Vite dev server, ESLint, PostCSS (likely), optionally Vercel config present.
- Testing: none obvious (no tests folder).

3) High-level src layout (approximate)
- `src/`
  - `assets/` - static images/fonts
  - `components/` - shared UI components (buttons, inputs, layout, header/footer, cards)
  - `pages/` or `screens/` - top-level pages (Home, Services, Booking, Dashboard, Auth)
  - `hooks/` - custom hooks (data fetching, auth, form helpers)
  - `services/` - API wrappers and network logic (calls to backend endpoints)
  - `store/` - client state (context, Redux, or simple hooks)
  - `utils/` - helpers and formatters
  - `routes/` or `App.jsx` - Router setup (React Router or similar)
  - `main.jsx` - app bootstrap

Note: Exact directories in this repo include `src/components`, `src/hooks`, `src/services`, `src/screens` or `src/pages` (follow those names when regenerating).

4) Key files to keep in mind
- `index.html` - Vite HTML template
- `vite.config.js` - Vite config
- `package.json` - dependencies + dev scripts (`dev`, `build`, `preview`, `lint`)
- `tailwind.config.js` - Tailwind config (theme customization may live here)
- `src/main.jsx` - React mount + global providers (theme, router, store)
- `src/App.jsx` - route definitions and layout wrapper
- `src/services/*` - API endpoints and auth token handling

5) Data flow and backend integration
- Frontend talks to backend at endpoints in `Event_backend/Server.js`. Common flows:
  - Auth & OTP: endpoints under `/api/otp` or `/otp`
  - Services list: `/api/services`
  - Booking creation: `/api/bookings` or similar
  - Reviews, Notifications, Subscriptions endpoints also exist
- Keep API base URL configurable via `.env` (there is a `.env` in this Frontend folder).

6) How the UI is structured (component responsibilities)
- Layout: `Header`, `Footer`, responsive container
- Pages compose smaller components: e.g., `ServicesPage` uses `ServiceCard`, `Filters`, `Pagination`
- Booking flow: multi-step form (select service -> choose slot -> enter customer details -> confirm)
- Dashboard: protected routes (check auth token) and shows bookings, revenue, reviews admin actions

7) Build & Run (commands you will need)
- Install:
  - `cd Frontend && npm install`
- Dev server:
  - `cd Frontend && npm run dev`
- Build:
  - `cd Frontend && npm run build`
- Preview production build:
  - `cd Frontend && npm run preview`
